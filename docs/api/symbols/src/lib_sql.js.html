1744          * be inverted, it throws an {@link patio.error.ExpressionError}.  An inverted expression should match

1759                 if (op == "AND" || op == "OR") {
1760                     newArgs = [OPERTATOR_INVERSIONS[op]].concat(expression.args.map(function(arg){

1778          *
1779          * <pre class='code'>

1781          * BooleanExpression.fromValuePairs({a : true}); // a IS TRUE;

1783          * </pre>

1786          * by default, OR possible).  If negate is set to true,
1787          * all subexpressions are inverted before used.
1788          * <pre class='code'>
1789          * BooleanExpression.fromValuePairs({a : [1,2,3], b : true}) //=> a IN (1,2,3) AND b IS TRUE
